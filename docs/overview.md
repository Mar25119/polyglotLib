# polyglot

## Введение: от англоцентризма к лингвистической демократии

Библиотека `polyglot` — это не просто инструмент для обработки естественного языка (NLP). Это про лингвистическую инклюзивность в мире технологий, где доминируют английский и несколько крупных языков. Созданная Абдельрахманом Бо Самуром (Abdelrahman Al-Omari) как часть исследовательского проекта в Университете Карнеги-Меллон, `polyglot` решает фундаментальную проблему современного NLP: **игнорирование 95% мировых языков**. В то время как большинство библиотек  сосредоточены на английском, немецком или китайском, `polyglot` поддерживает **более 190 языков**, включая редкие, малоизученные и даже умирающие — от суахили и тамильского до навахо и идиш.

Под капотом `polyglot` использует данные проекта **CLD2 (Compact Language Detector 2)** от Google и модели на основе **ICU (International Components for Unicode)**, чтобы обеспечить корректную обработку самых разных письменностей: латиницы, кириллицы, арабицы, китайских иероглифов, деванагари и многих других. Это делает его уникальным инструментом для проектов, где важна **глобальная репрезентативность**, а не только коммерческая выгода. Хотя библиотека больше не активно развивается (последний релиз — 2019 год), её наследие остаётся актуальным: **технологии должны служить всем людям, а не только носителям крупных языков**.

## Модуль 1. Контекст и предпосылки

### 1. Ограничения стандартных средств: тирания «больших» языков

Стандартные библиотеки NLP в Python (NLTK, spaCy) исторически ориентированы на **англоязычный мир**:
- Токенизация — по пробелам (не работает для китайского, японского, тайского);
- Распознавание сущностей — только для латиницы;;
- Поддержка языков — ограничена десятком «коммерчески выгодных».

Это создаёт **цифровое неравенство**: пользователи малых языков получают худший опыт, их контент игнорируется, а их культуры маргинализируются. Даже современные трансформеры часто плохо работают на языках с ограниченными данными.

### 2. Проблема, которую решает инструмент: NLP для всех языков мира

`polyglot` решает фундаментальную проблему **лингвистического разнообразия**. Он предоставляет базовые NLP-операции для любого языка, поддерживаемого ICU:
- **Определение языка** (Language Detection) — по тексту;
- **Токенизация** (Tokenization) — с учётом правил языка;
- **Транслитерация** — преобразование между письменностями.

Это особенно важно в следующих сценариях:
- **Международные соцсети**: автоматическая маршрутизация модерации;
- **Цифровые гуманитарные науки**: анализ исторических текстов;
- **Глобальные платформы отзывов**: обработка мнений на местных языках;
- **Образовательные проекты**: демонстрация лингвистического разнообразия.

### 3. Инженерная / визуальная грамотность: от данных к культуре

Использование `polyglot` формирует **культуру лингвистической инклюзивности**. Разработчик перестаёт думать: «Как обработать английский текст?», и начинает спрашивать: «Как система будет работать с текстом на любом языке?». Это сдвиг от **техноцентризма** к **культурной чуткости**.

Например, вместо:
```python
# Англоцентричный подход
words = text.split()  # Не работает для китайского!
```

— можно использовать:
```python
# Инклюзивный подход
from polyglot.text import Text
words = Text(text).words  # Работает для 190+ языков
```

Первый вариант — это игнорирование реальности, второй — уважение к ней.

### 4. Применение в реальных проектах: от исследований до модерации

`polyglot` используется в нишевых, но важных проектах:

- **Академические исследования**: лингвистика, антропология, история;
- **Многоязычная модерация**: Facebook, Twitter используют аналогичные системы;
- **Цифровые архивы**: обработка документов на колониальных или редких языках;
- **Образовательные платформы**: демонстрация структуры языков;
- **Гуманитарные миссии**: анализ сообщений в зонах конфликтов.

Особенно ценен в проектах, где **поддержка малых языков — этическая обязанность**, а не коммерческий выбор.

## Модуль 2. Основные идеи и механизмы

### 5. Центральные объекты и архитектура: ICU как основа

Архитектура `polyglot` построена на двух столпах:

- **CLD2**: для определения языка по тексту (до 83 языков);
- **ICU**: для токенизации, NER и других операций (190+ языков).

Ключевые компоненты:
- **`Detector`**: определяет язык и уровень уверенности;
- **`Text`**: предоставляет методы для токенизации, NER, анализа тональности;
- **Языковые модели**: загружаются автоматически при первом использовании (~100–300 МБ).

Пример:
```python
from polyglot.detect import Detector
from polyglot.text import Text

# Определение языка
lang = Detector("Bonjour le monde").language
print(lang.name, lang.code, lang.confidence)  # French fr 98.23

# Токенизация
words = Text("你好世界").words
print(words)  # ['你', '好', '世', '界']
```

### 6. Ключевые механизмы работы: адаптация под письменность

`polyglot` корректно обрабатывает особенности разных письменностей:

- **Китайский/японский**: токенизация без пробелов через словари ICU;
- **Арабский**: обработка связанных форм букв и направления письма;
- **Тайский**: разделение слоговых блоков;
- **Деванагари (хинди)**: обработка сложных лигатур.

Это достигается за счёт использования **языково-специфических правил** из ICU, а не универсальных эвристик.

### 7. Работа со структурированными данными: от текста к метаданным

Результат работы `polyglot` — это не просто строки, а **структурированные метаданные**:

```python
result = {
    "language": "Chinese",
    "code": "zh",
    "confidence": 98.72,
    "words": ["今", "天", "天", "气", "很", "好"],
    "entities": [("北京", "I-LOC")]  # если включён NER
}
```

Такие данные можно использовать для:
- Маршрутизации в системах модерации;
- Автоматической классификации контента;
- Лингвистического анализа.

### 8. Интерактивность / автоматизация: pipeline многоязычной обработки

`polyglot` легко интегрируется в автоматизированные pipeline:
1. Получение текста (соцсеть, мессенджер);
2. Определение языка;
3. Выбор обработчика по языку;
4. Токенизация, NER, анализ тональности.

Пример:
```python
def process_message(text):
    lang = Detector(text).language
    if lang.confidence > 80:
        doc = Text(text, hint_language=lang.code)
        return {
            "lang": lang.code,
            "words": doc.words,
            "sentiment": doc.polarity  # если модель загружена
        }
    else:
        return {"lang": "unknown"}
```

### 9. Ошибки и отладка: честное признание неопределённости

`polyglot` корректно обрабатывает неоднозначные ситуации:
- **Короткие тексты**: низкая уверенность (например, `"Hi"` → 60%);
- **Смешанные языки** (code-switching): может определить основной язык;
- **Редкие языки**: модель может отсутствовать (падает исключение).

Это **не ошибка библиотеки**, а честное признание: «Я не уверен». Такое поведение предпочтительнее, чем ложная уверенность.

### 10. Организация результата: читаемость, композиция, масштабируемость

Код с `polyglot` обладает ключевыми качествами исследовательского ПО:
- **Читаемость**: каждая операция явна;
- **Композиция**: легко комбинировать с другими библиотеками (например, Hugging Face);
- **Масштабируемость**: поддержка 190+ языков «из коробки».

Пример расширения:
```python
# Было: только токенизация
words = Text(text).words

# Стало: токенизация + NER
doc = Text(text)
entities = [(e, e.tag) for e in doc.entities]
```

## Модуль 3. Анализ и оценка

### 11. Сильные стороны: почему polyglot уникален

- **Поддержка 190+ языков**: рекорд среди open-source NLP-библиотек;
- **Открытые модели**: нет зависимости от коммерческих API;
- **Корректная работа с Unicode**: все письменности обрабатываются правильно;
- **Простота использования**: базовые операции в 2–3 строках кода;
- **Академическая ценность**: инструмент для исследований.

### 12. Ограничения и недостатки: границы применимости

- **Установка**: требует ICU, что сложно на Windows;
- **Отсутствие поддержки**: последний релиз — 2019 год;
- **Потребление ресурсов**: модели занимают сотни МБ;
- **Ограниченная функциональность**: нет современных моделей (трансформеров);
- **Низкая точность для редких языков**: из-за нехватки обучающих данных.

### 13. Личное мнение: важно для исследований, с осторожностью для продукции

`polyglot` — это не просто библиотека, а **этическая позиция**: технологии должны быть инклюзивными. Я убеждён, что любой проект, работающий с многоязычным контентом, **обязан рассмотреть `polyglot`** как часть своего стека — даже если в продукции будет использоваться другое решение.

Проект «Многоязычный анализ текста» — маленький, но он демонстрирует суть: **уважение к каждому языку — это основа глобальных технологий**. Именно такие подходы делают разработку по-настоящему человечной.
